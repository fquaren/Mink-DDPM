# OPERA Precipitation Super-Resolution Data Pipeline

This repository contains the data engineering pipeline for training conditional diffusion models on high-resolution OPERA radar precipitation data. The pipeline transforms raw, strictly positive physical fields into memory-efficient, mathematically bounded tensors optimized for deep learning. Specifically, it resolves the class imbalance inherent to atmospheric variables and aligns the empirical data distribution with the theoretical prior of the diffusion Markov chain. 

## Pipeline Architecture

The preprocessing workflow is divided into three consecutive disk-based stages, orchestrated via Slurm, followed by a dynamic PyTorch data loader at runtime.

### Stage 1: Spatial Profiling and Metadata Generation (`generate_metadata.py`)

Because precipitation fields are heavily zero-inflated, loading full multidimensional arrays to locate convective systems creates a severe I/O bottleneck. 

* **JIT Compilation:** Uses a Numba JIT-compiled loop to scan raw daily Zarr stores efficiently.
* **Spatial Masking:** Identifies valid spatial patches (free of `NaN` values) matching the configured patch size.
* **Intensity Tracking:** Extracts the maximum physical precipitation intensity ($I_{max}$) for each patch and writes it directly to the metadata file. This isolates the physical state (wet vs. dry) without requiring downstream scripts to open the heavy data arrays.

### Stage 2: Stochastic Shuffling and Splitting (`consolidate_and_split_shuf.py`)

To ensure statistical homogeneity across training epochs, the daily metadata is temporally and spatially decorrelated.

* **Memory Efficiency:** Utilizes highly optimized Linux command-line tools (`cat`, `shuf`, `wc`) to process millions of metadata rows entirely in RAM or minimal swap space, bypassing Python's memory overhead.
* **Partitioning:** Splits the shuffled dataset into `train`, `validation`, and `test` text files based on strict configuration ratios.

### Stage 3: Physical Transformation and Zarr Consolidation (`preprocess_data.py`)

This script executes the physical transformations and stores the final machine learning-ready arrays.

* **Drizzle Filtering:** Enforces exact zeros by applying a background noise threshold (e.g., 0.1 mm/h).
* **Mass Conservation:** Generates the low-resolution condition via block averaging (mean pooling), physically conserving the total water mass within the grid cell.
* **Interpolation:** Generates the bicubically interpolated condition. It applies a strict zero-lower-bound clip to eliminate unphysical negative values generated by spline ringing artifacts.
* **Consolidation:** Writes the physical arrays into a singular, chunked `preprocessed_dataset.zarr` store. The chunks are aligned exactly to the patch dimensions to maximize read throughput during training.
* **Global Scaling:** Computes the global maximum of the $\log(1+x)$ transformed training data, saving it as `log_precip_max_val.npy` for downstream domain shifting.

### Stage 4: Topological Target Extraction (compute_gamma_targets.py)

This script computes geometric and topological metrics (area, perimeter, Euler characteristic) to enforce macroscopic physical constraints during diffusion training via a Minkowski (1-Wasserstein) loss.

* **Empirical CDF Extraction:** Randomly samples millions of wet pixels from the training split to estimate the empirical cumulative distribution function. It converts user-defined probability quantiles (q) into exact physical precipitation thresholds (τ in mm/h).
* **Topological Computation:** Uses `gudhi` and `skimage` to construct cubical complexes and extract the morphological features of the precipitation fields across the derived physical thresholds.
* **Zarr Integration:** Appends the resulting tensors directly into the preprocessed_dataset.zarr store as a gamma_targets array, aligning perfectly with the spatial chunks to eliminate I/O bottlenecks during model training.

### Stage 5: Dynamic Domain Shifting and Balancing (`dataset.py`)

The PyTorch `Dataset` handles the final mathematical transformations dynamically during the training loop.

* **Class Balancing:** Parses the metadata $I_{max}$ column. Randomly subsamples the dry background patches to achieve a specified ratio (e.g., 1:1 wet-to-dry) to prevent statistical mode collapse.
* **Multiprocessing Safety:** Uses a lazy Zarr initialization to prevent file descriptor deadlocks when PyTorch spawns multiple CPU workers.
* **Domain Shift:** Applies the logarithmic transform, normalizes by the global scaler, and shifts the domain to $[-1, 1]$. This affine transformation is mathematically required to align the initial data distribution with the $\mathcal{N}(0, I)$ prior of the diffusion model.

## Physical and Mathematical Constraints

1. **Atmospheric Sparsity:** Precipitation data is fundamentally non-Gaussian and zero-inflated. The pipeline explicitly tracks wet/dry regimes to ensure the neural network is exposed to sufficient convective events, rather than minimizing loss against trivial clear-sky backgrounds.
2. **Diffusion Model Alignment:** Standard diffusion mechanisms degrade if the initial data mean is heavily skewed. By strictly shifting the domain to $[-1, 1]$, we eliminate systematic gradient drift, allowing the U-Net to learn structural score functions efficiently.
3. **Topological Fidelity:** Translating probability quantiles into strict physical intensity thresholds ensures that the Minkowski loss correctly regularizes the extreme convective cores based on the true climatological distribution, rather than unphysical arbitrary cutoffs.

The entire preprocessing pipeline is orchestrated via a Slurm batch script.

1. Configure the environmental variables, chunk sizes, and thresholds in `configs/config.yaml`.
2. Submit the pipeline to the cluster:

```bash
sbatch scripts/run_preprocessing_pipeline.sh
